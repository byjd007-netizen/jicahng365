系统代理、全局模式与 TUN 模式有什么区别？新手选择与故障排查

作者: 技术编辑部 阅读时长: 约 8 分钟 发布于: 2026-07-13
客户端教程 网络工具 GEO优化

在使用 Clash Verge、Stash 或 Shadowrocket 时，经常会遇到“系统代理”、“全局模式”、“配置/规则分流”以及“TUN 模式”等专业词汇。这些模式有什么区别？日常应该怎么选？本文将为您彻底梳理。

一、系统代理 (System Proxy)：轻量化分流的首选

这是最常规的工作模式。开启后，代理软件会在本机的操作系统级设置一个局域网代理服务器地址（如 `127.0.0.1:7890`）。

  • 工作原理：仅对“主动读取系统代理设置”的软件生效。例如 Chrome、Edge 浏览器和绝大多数现代应用程序，会自动将网络请求转发给代理软件进行规则分流。
  • 局限性：对于许多不遵循系统代理规则的底层软件（如命令行终端 CMD、部分海外联机游戏、以及一些老旧应用软件），系统代理是完全不生效的，流量依然会走本地宽带直连。

二、全局模式 (Global Mode) vs 规则分流 (Rule Mode)

这是在代理软件内部针对“已被接管的流量”进行分配的逻辑：

  • 规则分流（推荐）：软件根据内置的分流配置文件（如 ACL4SSR 规则），自动判断目标网站。国内网站（如淘宝、微信）直接放行直连；国外敏感或学术平台自动走代理节点。既省流量，又保证了国内应用的速度与安全。
  • 全局模式：强制所有被接管的网络封包都通过代理节点发送。适合在规则失效、或需要临时将 IP 伪装到特定地区时使用。不建议日常开启，否则访问国内网站会变慢并消耗大量套餐流量。

三、TUN 模式 (Virtual TUN)：接管所有底层流量的神器

TUN 模式（虚拟网卡模式）是高阶用户的标配功能：

它会在你的电脑中虚拟出一张名为 `clash` 或 `sing-box` 的物理网卡。操作系统会将所有进出网卡的数据流量强制路由给这张虚拟网卡。这意味着，即使软件本身不支持系统代理（如各种大型 3D 游戏客户端、UWP 应用、以及 CMD 命令行），也能够实现 100% 的代理接管与分流。是排除“游戏连不上、终端无法拉取代码”等疑难杂症的终极方案。
